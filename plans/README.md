## Direction options (not planned — for the maintainer to weigh)

Two grounded strategic directions surfaced by the audit, deliberately NOT turned into plans (they are design spikes, not mechanical work — say the word and a spike plan gets written):

- **A. First-class artifact UPDATE contract.** "update this HTML" is an advertised trigger (`SKILL.md:4`), but the whole update contract is two sentences (`SKILL.md:21`, `:382`) and `lint-artifact.mjs` is stateless/single-file — it cannot detect that an update dropped a working control, orphaned a localStorage key, or broke copy-as-prompt. Spike: an "Updating an existing artifact" SKILL section + a `lint-artifact --against <prior.html>` diff mode that warns on dropped ids/state keys.
- **B. Inline JSON source island for all shapes.** The podcast path already proves it (`episode-data` island → deterministic re-render); editorial computes rich intermediates (entities, themes, chapter summaries) and throws them away. An `application/json` island would make updates lossless and copy-as-prompt exact — at the cost of artifact size and extending the publish-time sensitivity scan to the island. Prototype on `editorial` first.
